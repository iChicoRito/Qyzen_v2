# Qyzen (Laravel) — Conventions

Tables keep the `tbl_*` names from the migrated Supabase schema so imported data and historical FK values line up (see `docs/roadmap/IMPLEMENTATION_ROADMAP_LARAVEL.md`). Eloquent models live in `app/Models` (one per table, each pinning its `$table = 'tbl_*'`); validation in Form Requests under `app/Http/Requests` (one per Zod schema); authorization in `app/Policies` plus Eloquent query scopes; domain/business logic in service classes under `app/Services`; and routes are split into one group per role (`admin`, `educator`, `student`, shared `profile`, public `auth`) — the Laravel equivalent of the Next.js role-based route groups. There is **no RLS**: every list query carries an ownership/enrollment scope and every route a Policy/middleware (authorization is explicit, everywhere). Timestamps store UTC; `correct_answer` is never serialized to a student.

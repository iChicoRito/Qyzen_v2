# Objective
## Quiz and Assessment Management Interface Refinements

---

## Role
You are a front-end developer and UI/UX specialist responsible for implementing interface improvements and dependency fixes in an educator-facing assessment platform.

---

## Description
This objective covers a series of targeted refinements to the educator interface for quizzes and assessments. The changes address redundant navigation, cross-subject question sharing, enhanced filtering capabilities, and modal layout adjustments. Each modification aims to streamline the educator workflow, reduce data redundancy, and improve overall usability. The work involves removing obsolete routes, enabling question reuse across different subject pools, expanding filter options, and repositioning interface elements for better user experience.

---

## Primary Objective
Refine the educator assessment and quiz interface by removing deprecated routes, implementing cross-pool question sharing, adding assessment code filtering, and adjusting modal button layouts.

---

## Secondary Objectives
- Remove the redundant `/educator/quizzes/create` page since quiz creation now occurs through a dedicated modal.
- Enable questions created in one subject pool to be borrowed or reused by other subject pools to prevent redundant data uploads.
- Add an assessment code filter to the `/educator/assessments` page alongside existing filters.
- Restructure the Manage Exemptions modal by removing the close button and repositioning the Exempt Selected and Un-exempt Selected buttons from left to right.
- Enhance the Question Bank filters on `/educator/quizzes` to make them independently selectable rather than displaying combined Code-SubjectName-Section entries.

---

## Success Criteria
- The `/educator/quizzes/create` route is fully removed and no longer accessible.
- Questions from one subject pool (e.g., Interactive Media Design) are accessible and usable in another subject pool (e.g., Graphics and Visual Computing) when topics or content overlap.
- A functional assessment code filter is present on the `/educator/assessments` page.
- The Manage Exemptions modal displays no close button, and the Exempt Selected and Un-exempt Selected buttons are aligned to the right side.
- The Question Bank filter area contains independent filters for Section, Subject, Assessment Code, and All Batches, with each filter contextually narrowing options based on prior selections.

---

## Constraints
- Remove only the `/educator/quizzes/create` page; the quiz creation modal must remain intact and functional.
- Do not alter the underlying data model for questions unless required for cross-pool sharing.

---

## Context & Dependencies
- A dedicated quiz creation modal already exists and serves as the primary method for creating quizzes, making the standalone `/educator/quizzes/create` page obsolete.
- Question pools are currently scoped to individual subjects, preventing reuse across subjects with identical topics.
- The `/educator/assessments` page currently filters only by All Assessments, Sections, and Statuses.
- The Manage Exemptions modal currently includes a close button and has action buttons positioned on the left.
- The Question Bank filter on `/educator/quizzes` currently presents subjects as a combined `Code-SubjectName-Section` string rather than independent filter dimensions.

---

## Supporting Tasks

### Route Cleanup
- [Tag: Sequential] Remove the `/educator/quizzes/create` route from the application's routing configuration.
- [Tag: Sequential] Delete the associated page component and any dead code references.
- [Tag: Sequential] Verify that all navigation links pointing to the removed route are updated or removed.

### Cross-Pool Question Sharing
- [Tag: Sequential] Modify the assessment pool logic so that questions are not strictly scoped to the subject where they were created.
- [Tag: Sequential] Implement a mechanism for educators to browse and select questions from other subject pools when both subjects share overlapping topics or content.
- [Tag: Sequential] Ensure the system prevents actual data duplication while allowing references to existing questions across pools.

### Assessment Code Filter
- [Tag: Sequential] Add an Assessment Code filter dropdown or input to the `/educator/assessments` page.
- [Tag: Sequential] Integrate the new filter with the existing filtering logic alongside All Assessments, Sections, and Statuses.

### Manage Exemptions Modal Adjustment
- [Tag: Sequential] Remove the close button from the Manage Exemptions modal.
- [Tag: Sequential] Reposition the Exempt Selected and Un-exempt Selected buttons from the left side to the right side of the modal footer.

### Question Bank Filter Restructuring
- [Tag: Sequential] Replace the combined `Code-SubjectName-Section` filter with independent filter controls.
- [Tag: Sequential] Implement separate filters for Section, Subject, Assessment Code, and All Batches.
- [Tag: Conditional] When a specific question or section is selected, cascade the available options in the remaining filters so that Subject only shows subjects belonging to the selected Section, and other filters behave similarly.

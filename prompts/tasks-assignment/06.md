# Objective
## Questions Bank, Assessment, Scores, Student Assessment, and Navbar UI Enhancements

---

## Role
You are a full-stack web developer responsible for improving the educator and student interfaces of a deployed assessment platform. Approach this objective with a focus on performance optimization, dynamic data filtering, and user experience refinement.

---

## Description
This objective encompasses several independent, concurrently executable improvements across the educator and student sides of the application. The work focuses on making filter dropdowns context-aware and dynamic, improving data management with bulk operations, and optimizing performance to prevent system slowdowns in the deployed environment. Specific user experience enhancements include modernizing time and date input methods and adding a secure deletion confirmation step for sensitive data. Finally, a minor user interface cleanup is required on the navigation bar. All tasks are designed to be executed simultaneously by separate sub-agents working independently.

---

## Primary Objective
Refine and optimize the assessment platform by implementing dynamic, context-aware filters across multiple pages, adding bulk deletion and secure score deletion, improving input UX, optimizing data loading with pagination, enforcing student assessment expiration, and cleaning up the navigation bar.

---

## Secondary Objectives
- Make filter dropdowns dynamically update to show only options tied to the current selection on the Questions Bank, Assessment, and Scores pages.
- Implement a multiple-deletion feature for the Questions Bank with a "select all" checkbox.
- Remove the "Also Add To These Assessments (optional)" field from both manual and file-upload question creation.
- Improve the time and date selection fields in the Add Assessment modal to allow direct typing.
- Optimize the Assessment Pool page with pagination to prevent loading all data at once.
- Add a search filter above the existing filters on the Scores page.
- Add a secure delete functionality to the Scores page context menu, requiring password verification.
- Audit and fix the student assessment expiration logic to prevent access after the due date.
- Comment out the non-functional search icon in the navigation bar without deleting the code.

---

## Success Criteria
- Selecting a section filter updates the subject, assessment, and batch filters to show only related items on the Questions Bank page.
- A "Select All" checkbox and a bulk delete action are functional for filtered questions in the Questions Bank.
- The "Also Add To These Assessments (optional)" option no longer appears in manual and file-upload question creation.
- The same dynamic filtering logic works correctly on the Assessment and Scores pages.
- Time and date fields in the Add Assessment modal accept typed input.
- The Assessment Pool page loads data in pages instead of all at once, eliminating primary system slowdowns.
- A search filter is visible and functional above the filters on the Scores page.
- The Scores context menu includes a "Delete Score" option that triggers a password confirmation modal before deleting a score.
- Students are blocked from starting or continuing an assessment if the current time is past the set end date/time.
- The search icon is commented out in the navbar code and is no longer visible in the UI.

---

## Context & Dependencies
- The application is already deployed, so performance improvements, particularly in the Assessment Pool, are critical to prevent slowness.
- A shared modal component exists and must be used for the score deletion confirmation.
- The Questions Bank already has dropdown filtration that works properly.
- An assessment expiration dependency may not be working correctly and requires an audit.

---

## Agent Assignments
Each sub-agent works independently and simultaneously. The tasks are grouped into six deployable agent packages below.

---

## Agent 1: Questions Bank Filtering & Bulk Deletion

### Primary Objective
Enhance the Questions Bank page with dynamic, context-aware dropdown filters and a bulk deletion feature.

### Success Criteria
- When a section is selected, the subject, assessment, and batch dropdowns update dynamically to show only options tied to that section.
- A "Select All" checkbox is visible and selects all currently displayed or filtered questions.
- A bulk delete action successfully removes all selected questions.

### Supporting Tasks
- [Tag: Sequential] Implement dynamic filtering so that selecting a section updates the subject, assessment, and batch dropdowns to show only options tied to that section.
- [Tag: Parallel] Add a "Select All" checkbox to the questions table that selects all currently displayed or filtered questions.
- [Tag: Parallel] Implement a bulk delete action that removes all selected questions.

# Objective

## Improve Announcement Notifications, Student Notification UI, and Rich-Text Formatting

---

## Role

You are a full-stack software developer responsible for reviewing, fixing, and improving the announcement, notification, and rich-text editor functionality across the educator and student interfaces.

---

## Description

Review the existing announcement workflow to ensure that announcements posted by educators generate and display notifications correctly on the student side. Improve the student notification layout by presenting the educator’s role as a soft badge and repositioning the notification timestamp for a cleaner interface. Replace the partially functioning rich-text editor with Quill.js to provide reliable formatting controls. Ensure that all formatting applied to an educator’s announcement is preserved and rendered consistently when students view the announcement.

---

## Primary Objective

Ensure that educator announcements, notifications, and rich-text formatting function correctly and display consistently across the educator and student interfaces.

---

## Secondary Objectives

* Verify and fix the notification flow for announcements posted by educators.
* Improve the layout and information hierarchy of student-side announcement notifications.
* Replace the existing rich-text editor with Quill.js.
* Preserve and display educator-created announcement formatting on the student announcement page.

---

## Success Criteria

* An announcement posted by an educator generates a notification that is visible to the intended students.
* Any notification-related defect found during verification is fixed.
* The student notification layout displays an **Educator** soft badge instead of showing the timestamp beneath or beside the educator’s name.
* The timestamp appears on the same row as the educator’s name and is aligned to the far-right side.
* The existing rich-text editor is replaced with Quill.js.
* The rich-text editor provides working formatting options beyond bold text.
* Announcement formatting created by educators is preserved and rendered correctly on the student announcement page.

---

## Constraints

* Use the Quill.js library for the rich-text editor instead of rebuilding the editor from scratch.

---

## Context & Dependencies

* The system contains separate educator and student interfaces.
* Educators can create and publish announcements.
* Students have a notification interface and an announcement page.
* The current rich-text editor is partially functional, with only the bold option working correctly.
* Quill.js documentation and implementation are available through `https://quilljs.com/`.

---

## Stakeholders

* **Educators:** Create announcements and apply rich-text formatting.
* **Students:** Receive announcement notifications and view formatted announcements.

---

## Supporting Tasks

### Agent 1 — Announcement Notification Verification

* [Tag: Parallel] Review the announcement workflow and verify whether an announcement posted by an educator generates a notification on the student side.
* [Tag: Conditional] If the notification already appears correctly, retain the existing behavior.
* [Tag: Conditional] If the notification does not appear, identify and fix the issue so educator announcements are included in the students’ notifications.

### Agent 2 — Notification UI and Announcement Formatting

* [Tag: Parallel] Update the student-side notification layout by removing the relative timestamp text, such as “5 minutes ago,” from its current position.
* Replace the removed text with a soft badge displaying the user role or user type as **Educator**.
* Display the timestamp on the same row as the educator’s name and align it to the far-right side of the row.
* Replace the existing rich-text editor with Quill.js because the current editor’s formatting options do not function correctly.
* Configure the Quill.js editor to provide multiple working rich-text formatting options.
* Ensure that the formatting applied to an announcement by an educator is preserved and rendered identically on the student announcement page.

---

## Detailed Breakdown

### Announcement Notification Flow

Inspect the complete announcement notification process, beginning when an educator publishes an announcement and ending when the notification is displayed to students. Confirm that the notification is created, associated with the correct recipients, and shown in the student notification interface. If any part of this flow is missing or malfunctioning, correct the implementation so educator announcements reliably appear in student notifications.

### Student Notification Layout

Retain the educator’s name in the notification item. Replace the relative timestamp currently displayed near the name with a soft badge labeled **Educator**. Move the timestamp to the far-right side of the same row as the educator’s name to improve visual organization and readability.

### Quill.js Rich-Text Editor

Remove or replace the current partially functioning rich-text editor implementation with Quill.js. Ensure that the editor’s available formatting controls operate correctly and are usable when educators create or edit announcements. Do not rebuild the editor manually when Quill.js can provide the required functionality.

### Announcement Formatting Consistency

Store and retrieve the educator’s formatted announcement content without losing its structure or styling. When the announcement is displayed on the student announcement page, render the content so that its formatting matches what the educator created in the rich-text editor.

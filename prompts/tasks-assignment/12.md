# Objective

## Resolve Learning Material Download Failures and Improve Assessment and Notification UX

---

## Role

You are a full-stack software developer responsible for diagnosing deployment-related file-access issues, improving asynchronous user interactions, and maintaining secure user-specific notification operations.

---

## Description

Investigate and resolve the recurring issue where downloadable student learning materials return a `404 Not Found` error after new code updates are pushed through Git, even though the files still exist. Improve the educator assessment workflow so granting special access and managing exceptions no longer reloads or redirects the entire page after submission. Preserve the existing functionality and business logic while enhancing only the user experience of these interactions. Add a user-scoped option for deleting all notifications on both the educator and student sides without affecting notifications belonging to other users.

---

## Primary Objective

Fix the learning material download issue and implement non-reloading assessment and notification enhancements while preserving existing functionality and user-data isolation.

---

## Secondary Objectives

* Prevent valid learning materials from becoming inaccessible after Git-based deployments or code updates.
* Allow educators to grant students special assessment access without reloading or leaving the current page.
* Allow educators to manage assessment exceptions without reloading or leaving the current page.
* Add a “Delete All Notifications” action for both educators and students.
* Ensure notification deletion affects only the currently authenticated user.

---

## Success Criteria

* Existing student learning materials remain downloadable after new code is pushed and deployed through Git.
* Valid learning material download requests no longer return a `404 Not Found` response because of incorrect or changed file paths.
* Granting special assessment access submits successfully without refreshing or redirecting the entire page.
* Managing assessment exceptions submits successfully without refreshing or redirecting the entire page.
* Existing special-access and exception-management logic continues to behave as it did before the UX changes.
* A “Delete All Notifications” control appears beside the existing “Mark All as Read” control.
* Educators can delete all notifications associated with their own accounts.
* Students can delete all notifications associated with their own accounts.
* Deleting notifications for one user does not delete or modify notifications belonging to any other user.

---

## Constraints

* Preserve the existing special-access functionality and business logic.
* Preserve the existing manage-exceptions functionality and business logic.
* Change only the submission experience for special access and exception management so the entire page does not reload.
* Notification deletion must always be restricted to the currently authenticated user.

---

## Context & Dependencies

* Learning material downloads currently work until new code updates are pushed through Git, after which some existing files return a `404 Not Found` response.
* The issue may be related to file-path handling or paths changing during deployment, but the implementation must also investigate and address any other related cause discovered.
* The educator assessment page currently reloads or redirects after granting a student special access.
* The manage-exceptions workflow currently reloads the entire page after submission.
* Both educator and student notification interfaces already include a “Mark All as Read” action.
* Notifications are associated with individual users and must remain isolated by account.

---

## Stakeholders

* Educators who manage assessment access, exceptions, and notifications.
* Students who download learning materials and manage their notifications.

---

## Supporting Tasks

### Learning Material Download Reliability

* Reproduce the learning material `404 Not Found` issue after a Git-based code update or deployment.
* Inspect how downloadable learning material paths are generated, stored, resolved, and accessed before and after deployment.
* Determine whether deployment updates are replacing, invalidating, or incorrectly resolving existing file paths.
* Investigate any additional warning signs or related causes discovered during diagnosis.
* Implement a permanent fix that keeps existing learning materials accessible across future code deployments.
* Verify that previously uploaded learning materials and newly uploaded materials remain downloadable.

### Special Assessment Access UX

* Update the educator assessment page so granting a student special access submits without reloading the entire page.
* Keep the educator on the current assessment view after a successful submission.
* Preserve the existing validation, authorization, data-processing logic, and resulting access behavior.
* Update the relevant interface state after submission so the educator can immediately see that the action succeeded.

### Assessment Exception Management UX

* Update the manage-exceptions workflow so submissions occur without reloading the entire page.
* Keep the educator on the current page or interface state after submission.
* Preserve all existing exception-management functionality, validation, authorization, and business logic.
* Limit modifications to the user experience and submission behavior.

### User-Scoped Notification Deletion

* Add a “Delete All Notifications” action beside the existing “Mark All as Read” action.
* Implement the action on the educator notification interface.
* Implement the action on the student notification interface.
* Delete only notifications owned by or associated with the currently authenticated user.
* Prevent the operation from deleting or modifying notifications belonging to other educators or students.
* Refresh the notification interface after successful deletion without exposing another user’s notification data.

### Verification and Regression Testing

* Test learning material downloads before and after a simulated Git-based deployment update.
* Test special-access submissions without a full-page reload.
* Test manage-exceptions submissions without a full-page reload.
* Confirm that both workflows still produce the same backend results as their existing implementations.
* Test notification deletion using multiple educator and student accounts.
* Confirm that deleting all notifications from one account leaves every other account’s notifications unchanged.

## Detailed Breakdown

### Learning Material Path and Deployment Investigation

Trace the complete learning material download flow, including how the file location is saved, how the downloadable URL is constructed, and how the application resolves the physical file after deployment. Compare this behavior before and after pushing updated code through Git. Correct any path-handling or deployment behavior that causes an existing file to become unreachable while ensuring that the fix remains reliable for future deployments.

### Non-Reloading Special-Access Submission

Modify the special-access interaction so the educator can grant access while remaining on the current assessment page. The request must still execute the same existing backend operation, authorization checks, validation rules, and data updates. Only the page-reloading or redirecting behavior and the related interface feedback should be improved.

### Non-Reloading Exception Management

Apply the same user-experience improvement to the manage-exceptions workflow. Submitting an exception must no longer reload the entire page, but it must continue to use the existing functional behavior and business rules. Update the visible interface state after submission so the educator receives clear confirmation without losing their current context.

### Secure Bulk Notification Deletion

Place the “Delete All Notifications” action beside “Mark All as Read” on both notification interfaces. The backend deletion operation must derive the affected notifications from the authenticated account and restrict the deletion query accordingly. An educator or student must never be able to delete another user’s notifications, even through a modified or manually crafted request.

### Final Integration Validation

Verify the changes together after integration. Confirm that asynchronous assessment actions remain stable, notification ownership is strictly enforced, and learning material downloads continue to work after deployment updates. Ensure that no unrelated functionality or existing business logic has been altered.

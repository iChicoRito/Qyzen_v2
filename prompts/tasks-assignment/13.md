# Objective
## Preserve Student Scores, Improve Assessment Filtering, Refine Score Modal Spacing, and Enhance Learning Material Management

---

## Role

You are a full-stack web developer responsible for maintaining data integrity, correcting educator-side interface issues, and improving assessment and learning-material management workflows.

---

## Description

Review and enhance several features on the educator side of the application. Ensure that deleting questions from the question bank does not remove or invalidate previously recorded student scores. Correct the student scores modal layout, resolve batch filtering so it works across the complete assessment question dataset rather than only the currently displayed page, and add bulk deletion support for uploaded learning materials. Ensure that a learning material uploaded for multiple sections or subjects uses one shared file instead of creating duplicate physical file copies.

---

## Primary Objective

Improve the educator-side question bank, student score modal, assessment filtering, and learning-material management features while protecting historical student results and preventing duplicate learning-material files.

---

## Secondary Objectives

- Verify and protect student scores when questions are deleted from the question bank.
- Correct the spacing between the student scores table and the modal footer.
- Ensure assessment questions can be filtered by batch regardless of the currently displayed pagination page.
- Add checkbox-based bulk deletion for uploaded learning materials.
- Allow one uploaded learning-material file to be shared across multiple selected sections or subjects without duplicating the physical file.

---

## Success Criteria

- Deleting one or all questions from the question bank does not delete or remove previously recorded student scores.
- Existing student results remain available even when their related question-bank entries are deleted.
- The student scores table has appropriate spacing from the modal footer and no longer appears visually crowded.
- Selecting a batch from the assessment question filter displays matching questions from the complete dataset.
- Batch filtering is not limited to questions loaded on the current pagination page.
- Filtering and pagination work correctly together.
- The learning-material table includes an individual checkbox for each material.
- The first column includes a Select All checkbox.
- A Bulk Delete button appears beside the Upload button when one or more materials are selected.
- Bulk deletion removes only the selected learning materials.
- Uploading one learning material for multiple sections or subjects stores only one physical file.
- All selected sections or subjects reference and share the same uploaded file.
- The shared learning material remains accessible from every assigned section or subject without creating duplicate file copies.

---

## Constraints

- Previously recorded student scores must not be deleted when question-bank questions are removed.
- Existing assessment functionality must remain operational while the filtering behavior is corrected.
- The Bulk Delete button must appear only when at least one learning material is selected.
- Bulk deletion must affect only the learning materials selected by the educator.
- Assigning a learning material to multiple sections or subjects must not create duplicate physical files.
- The same stored file must be shared through its associations with the selected sections or subjects.

---

## Context & Dependencies

- The educator side contains a question bank where questions can be deleted.
- Student scores may currently depend on records associated with question-bank entries.
- Student scores are displayed inside a modal containing a table and footer.
- Assessment questions use pagination and include an All Batches dropdown filter.
- The current batch filter appears to process only the questions displayed on the active pagination page.
- The educator-side learning-material section already includes an Upload button and supports individual materials.
- A learning material may be uploaded and assigned to multiple sections or subjects.
- The current upload behavior may create duplicate file copies when multiple sections or subjects are selected.

---

## Stakeholders

- **Educators:** Manage question-bank entries, review student scores, filter assessment questions, and upload, assign, or delete learning materials.
- **Students:** Depend on the system to preserve their historical assessment scores and provide access to learning materials assigned to their sections or subjects.

## Detailed Breakdown

### Question Deletion and Historical Scores

Deleting question-bank entries must not remove previously recorded student scores. Review whether score records, submitted answers, or completed assessment results are deleted through cascading database relationships or question-deletion logic. Adjust the behavior so question-bank maintenance remains separate from historical student performance data.

### Student Scores Modal Spacing

The table inside the student scores modal currently appears too close to the modal footer. Adjust the table container, modal body, or footer spacing so there is clear visual separation without changing the modal’s existing functionality.

### Dataset-Level Batch Filtering

The current batch filter appears to operate only on assessment questions already loaded on the active page. Change the workflow so the selected batch is applied to the complete question dataset before pagination is calculated. This must allow educators to find matching questions even when those questions would otherwise appear on another pagination page.

### Bulk Material Selection

Place a Select All checkbox in the first column header and an individual checkbox in the first column of every material row. When the educator selects one or more learning materials, display the Bulk Delete button beside the existing Upload button. Submitting the bulk action must delete only the selected records and then update the displayed material list.

### Shared File Assignment

When an educator uploads a learning material and assigns it to multiple sections or subjects, the system must upload and store the physical file only once. Each selected section or subject must reference that same stored file rather than generating a separate duplicate copy. Removing the material from one assignment must not make the file unavailable to other sections or subjects that still use it.

# Objective
## Streamline Grant and Exemption Access Management UX

---

## Role
You are a UX designer and front-end developer focused on reducing manual steps in administrative workflows. Approach this objective with a mindset centered on simplicity, automation, and minimizing user interaction friction.

---

## Description
The current process for managing student grant access and exemptions is overly manual and repetitive. When a student uses their grant access, the system does not automatically reflect that usage, forcing administrators to manually revoke access before reassigning it to other students. This creates a cumbersome loop of revoking, reopening, and reassigning that wastes time and introduces unnecessary steps. The goal is to redesign this interaction so that grant usage automatically unchecks or removes that student from the active access list. Additionally, the exemption management interface uses confusing toggle-like button labels that require separate actions for granting and revoking, when a simple save-based interaction would suffice.

---

## Primary Objective
Redesign the grant access and exemption management interfaces so that changes are handled automatically upon usage and consolidated into a single "Save Changes" action, eliminating the need for separate grant/revoke and exempt/un-exempt workflows.

---

## Secondary Objectives
- Automatically uncheck students who have used their grant access so they do not need to be manually revoked before adding new students.
- Replace the dual-button pattern (Grant Selected / Revoke Selected and Exempt Selected / Un-exempt Selected) with a single "Save Changes" interaction where only checked items are affected and unchecked items are ignored.

---

## Success Criteria
- When a student uses their grant access, that student is automatically unchecked from the active access list without manual intervention.
- The interface displays a single "Save Changes" action instead of separate Grant/Revoke and Exempt/Un-exempt buttons.
- Saving changes applies only to checked items; unchecked items remain in their current state without any modification.

---

## Constraints
- The word "Exempt Selected / Un-exempt Selected" and "Grant Selected / Revoke Selected" must be replaced with "Save Changes."
- When saving, unchecked items must not be affected; only checked items should trigger a state change.

---

## Out of Scope
- Changes to the underlying permission or grant logic beyond the checkbox and save interaction.
- Modifications to how grants or exemptions are initially assigned outside of the manual student selection screen.

---

## Stated Assumptions
- The current system uses a checkbox-based interface for selecting students to grant or revoke access and exemptions.
- Grant access usage can be detected by the system and should trigger an automatic uncheck action.

---

## Supporting Tasks

### Grant Access Automation
- Implement a trigger that automatically unchecks a student from the grant access list when that student uses their grant.
- Ensure the automatic uncheck occurs without requiring a separate revoke step by the administrator.

### UI Label and Interaction Consolidation
- Remove the "Grant Selected" and "Revoke Selected" button pair.
- Remove the "Exempt Selected" and "Un-exempt Selected" button pair.
- Add a single "Save Changes" button that applies state changes only to checked students.
- Ensure that when "Save Changes" is clicked, unchecked students are ignored and remain in their current state, while checked students have the corresponding action applied.

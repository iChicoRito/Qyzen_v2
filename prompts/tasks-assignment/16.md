# Objective
## Offline Score Upload, Assessment Submission, and Enrollment Management Enhancements

---

## Role
You are a full-stack web developer responsible for implementing user interface improvements, backend logic updates, and asynchronous request handling.

---

## Description
The objective is to enhance several existing modules within an educational platform. The offline score upload interface must be updated to include a term selection dropdown and simplified input fields, while visually matching the design language of other upload features. The assessment submission process must be refactored to use AJAX, eliminating page reloads on form submission while preserving all current functionality. The date and time picker components require integration of a JavaScript library to replace manual scrolling with a proper selection interface. The enrollment management page needs two additions: a cancel button for queued operations within the recent uploads timeline, and a structured download feature that exports all enrolled student data into a zip archive containing organized folders and worksheet files, mirroring the existing score download flow.

---

## Primary Objective
Implement targeted UI/UX enhancements across offline score upload, assessment submission, date/time selection, and enrollment management modules.

---

## Secondary Objectives
- Add a term dropdown to the offline score upload to distinguish scores when multiple terms exist.
- Simplify the offline score upload form so only student number and scores are required, without question count.
- Replicate the header, color, and layout design of existing upload features for the offline score upload.
- Convert the assessment page submission from file upload with page reload to an AJAX-based submission that preserves all current functionality without reloading.
- Integrate a JavaScript library for date and time pickers to enable easy selection without manual scrolling.
- Add a "Cancel" button to the recent enrollment uploads timeline to cancel the current queue.
- Add a "Download Enrollment" button that exports all enrolled student data as a zip archive with organized folders and worksheet files, following the same structure and flow as the existing score download process.

---

## Constraints
- Offline score upload must retain the visual design (headers, colors, etc.) consistent with user upload, enrollment upload, and question upload features.
- Assessment page submission must retain existing functionality; only the submission method changes to AJAX.
- Enrollment download must mirror the organization and file/folder structure of the existing score download process.

---

## Supporting Tasks

### Offline Score Upload Enhancements
- Add a dropdown selector for "Term" to the offline score upload form so users can assign scores to a specific term when multiple terms exist.
- Remove the question count input field from the offline score upload form, requiring only student number and scores.
- Apply the header, color, and layout styles from existing upload features to the offline score upload interface.

### Assessment Submission Refactor
- Replace the current file upload submission method with an AJAX request on the assessment page.
- Ensure the page does not reload upon submission while all existing functionality remains intact.

### Date and Time Picker Improvement
- Integrate a JavaScript library that provides an intuitive date and time picker, replacing the current manual scrolling selection.

### Enrollment Management Additions
- [Sequential] Add a "Cancel" button to the recent enrollment uploads timeline that cancels the currently queued operation.
- [Sequential] Add a "Download Enrollment" button that exports all enrolled students for the educator into a zip archive.
- Structure the zip archive with separate folders and worksheet files, organizing student credentials including number, name, email, section, and subject, following the same flow and organization as the existing score download feature.

---

## Multi-Agent Feasibility Assessment

**Assessment:** YES. The tasks span multiple independent modules — offline score upload UI, assessment AJAX submission, date picker integration, and enrollment management features — each with minimal cross-dependencies. They can be developed in parallel by separate sub-agents.

**Parallelization Opportunities:**
- Offline score upload UI changes (term dropdown, field reduction, design alignment)
- Assessment page AJAX submission refactor
- Date and time picker library integration
- Enrollment management features (cancel button and download enrollment)

**Dependencies to Manage:**
- Enrollment download must reference and replicate the existing score download flow, requiring coordination with that module’s structure.
- Offline score upload design consistency depends on alignment with existing upload feature styles.

**Named Sub-Agent Assignments:**
- **Score Upload UI Agent** (Frontend Developer): Implement term dropdown, simplify form fields, and apply consistent design to the offline score upload page.
- **Assessment AJAX Agent** (Full-Stack Developer): Refactor the assessment submission process to use AJAX without altering functionality.
- **Date Picker Integration Agent** (Frontend Developer): Integrate a JavaScript library for proper date and time selection across the application.
- **Enrollment Management Agent** (Full-Stack Developer): Add the cancel queue button and implement the structured zip download for enrollment data, mirroring the score download flow.

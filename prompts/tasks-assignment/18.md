# Objective
## Comprehensive Audit and Remediation of Student-Facing Production Errors

---

## Role
You are a full-stack web application debugger and Laravel specialist. Approach this objective with the mindset of a production-support engineer, prioritizing root-cause analysis, secure authorization logic, and user-experience integrity for the student user role.

---

## Description
Conduct a thorough audit of all student-facing pages within the production Laravel application. Several distinct, critical errors have been reported by students during active exam sessions. The primary reported issue is an intermittent “403 | This action is unauthorized” error appearing on the quiz page while students are answering an exam, though the native Laravel error handler is not revealing the underlying exception. Additional bugs include non-functional hint buttons, specifically within the math minigame hint, and an auto-retake loop that triggers after a student completes an assessment. The objective is to investigate all reported errors, identify their root causes, and implement permanent fixes. Beyond these specific reports, proactively explore all student-side pages to uncover any additional latent issues that could disrupt the examination process. The desired outcome is a fully stable, error-free experience for every student interaction.

---

## Primary Objective
Audit, diagnose, and resolve all production errors and bugs on the student-facing side of the application, ensuring that every reported and potential issue is permanently fixed.

---

## Secondary Objectives
- Investigate and fix the intermittent “403 | This action is unauthorized” error occurring on the quiz page during exams.
- Repair non-functional hint buttons, with special attention to the math minigame hint.
- Resolve the erroneous auto-retake behavior that triggers after a student submits an assessment.
- Proactively scan all student pages to uncover and remediate any other potential errors or usability defects.

---

## Success Criteria
- Zero occurrences of the “403 | This action is unauthorized” error for students actively taking quizzes.
- All hint buttons, including those in the math minigame, function correctly on every interaction.
- Students are no longer forced into an auto-retake after completing an assessment; the post-submission flow follows the intended logic.
- A full audit report is produced confirming that all student-side pages are free of critical bugs and authorization flaws.

---

## Constraints
- The errors occur in a production environment where native Laravel error pages are suppressed; diagnosis must rely on logs, non-invasive monitoring, or replicated conditions.
- All fixes must be implemented without disrupting the live exam experience for currently active students.

---

## Priority Level
- Critical: All identified errors and the proactive audit must be completed and verified as a top priority due to direct impact on active student exams and assessment integrity.

---

## Context & Dependencies
- The application is built on the Laravel framework and has custom authorization logic (policies/gates) governing quiz access.
- The quiz page includes dynamic features such as hints and a math minigame that rely on front-end interactivity and back-end services.
- The assessment workflow includes a post-submission process that evaluates answers and determines whether a retake is required or permitted.
- The production environment suppresses default Laravel error pages, requiring alternative debugging approaches.

---

## Supporting Tasks

### Initial Investigation and Logging Enhancement
- [Tag: Parallel] Inspect Laravel logs, failed-job records, and any available exception-tracking service for clues about the 403 error’s origin.
- [Tag: Parallel] Temporarily enhance logging around the authorization checks on the quiz controller and any related policy classes to capture detailed failure context without exposing errors to users.
- [Tag: Parallel] Review and augment client-side error tracking for the hint buttons and math minigame interactions to capture JavaScript console logs and failed network requests.

### Root-Cause Analysis and Targeted Fixes
- [Tag: Conditional — on identifying cause] Isolate the specific policy, gate, or middleware returning the 403 response during active quiz sessions and implement a permissions or session-state fix.
- [Tag: Conditional — on identifying cause] Debug the hint system’s front-end event bindings and back-end hint-delivery endpoints, focusing on the math minigame, and restore full button functionality.
- [Tag: Conditional — on identifying cause] Trace the post-assessment evaluation and retake-triggering logic to find the condition causing the automatic retake, then correct the workflow to respect the intended pass/fail or completion state.

### Proactive Student-Side Audit
- [Tag: Sequential — after targeted fixes verified] Perform a comprehensive walkthrough of every student page, including login, dashboard, quiz navigation, hint systems, minigames, and post-assessment screens, testing for authorization errors, broken controls, and state mismanagement.
- [Tag: Sequential — after audit] Document and resolve any newly discovered issues.

### Final Verification
- [Tag: Sequential] Confirm all fixes in a staging environment that mirrors production, then deploy and monitor in production for a defined observation window.

## Multi-Agent Feasibility Assessment
**Assessment:** YES. The reported issues span three distinct functional areas (authorization errors, broken UI controls, and a flawed post-submission workflow) plus a broad proactive audit. These work streams can be investigated in parallel by specialists focusing on back-end authorization, front-end interactivity, and workflow logic. A fourth agent can concurrently perform the proactive audit of remaining student pages. Each stream has clear independence with manageable integration points.

**Parallelization Opportunities:**
- Diagnose the 403 authorization error on the quiz page by analyzing back-end policies, middleware, and log data.
- Debug and fix the non-functional hint buttons and math minigame interactions at the front-end and API layer.
- Investigate the assessment auto-retake loop within the post-submission business logic.
- Conduct a proactive manual and automated scan of all student-facing pages for any other issues.

**Dependencies to Manage:**
- All agents must coordinate on a shared staging environment that replicates the production state to avoid conflicting code changes.
- Findings from the targeted fix agents should inform the proactive audit agent of common failure patterns to accelerate discovery.
- A final integration and regression-testing phase must merge all fixes and verify no cross-fix collisions.

**Named Sub-Agent Assignments:**

- **Agent Auth-Debugger** (Back-End Laravel Security Specialist): Investigate and fix the “403 | This action is unauthorized” error by auditing quiz-related policies, gates, middleware, and session management.
- **Agent UI-Debugger** (Front-End and API Integration Specialist): Debug and repair the non-functional hint buttons, with a focus on the math minigame hint, addressing both client-side event handling and back-end hint-delivery endpoints.
- **Agent Workflow-Debugger** (Business Logic and State Machine Specialist): Trace and correct the post-assessment logic that erroneously triggers an auto-retake after submission.
- **Agent Proactive-Auditor** (Full-Stack QA Specialist): Systematically walk through every student-facing page to uncover and catalog any additional errors, usability bugs, or authorization flaws not yet reported.

# Objective
## Redesign New User Registration Verification Flow

---

## Role

You are a UX-focused authentication system developer. Approach this objective with the mindset, vocabulary, and priorities of a developer responsible for designing secure, intuitive, and seamless user account activation workflows.

---

## Description

Redesign the account registration and verification process for newly created user accounts by replacing the existing temporary password verification flow with a six-digit email verification code. Instead of requiring users to verify their account and immediately use their email and temporary password, the new workflow should guide them through a dedicated verification step using a one-time code sent to their registered email address. The experience should provide a smooth and intuitive user interface, including automatic cursor progression between individual code input fields. Once the verification code is successfully validated, users should proceed directly to the existing forced password change process without altering that functionality.

---

## Primary Objective

Replace the current temporary password account verification process with a six-digit email verification code workflow that seamlessly transitions verified users into the existing forced password change process.

---

## Secondary Objectives

- Modify the registration email so it includes the user's email address and a six-digit access/verification code instead of a temporary password.
- Redirect users to a dedicated verification screen after they click the **Verify Account** button.
- Display a verification interface consisting of six individual input fields for the verification code.
- Improve the verification user experience by automatically moving the cursor to the next input field after each digit is entered.
- Preserve the existing forced password change functionality after successful account verification.

---

## Success Criteria

- Newly registered users receive an email containing their registered email address and a six-digit verification code.
- Clicking the **Verify Account** button redirects users to the verification page instead of completing verification immediately.
- The verification page displays six individual code input fields.
- Each entered digit automatically advances focus to the next field.
- Users can successfully verify their account using the correct six-digit code.
- After successful verification, users are redirected directly to the existing forced password change page.
- The existing forced password change functionality remains unchanged apart from the new entry point.

---

## Context & Dependencies

- The current registration workflow sends users their email address, a temporary password, and a **Verify Account** button.
- The system already includes an account verification process.
- The application already has an existing forced password change flow that should continue to be used after successful verification.
- The registration email template and verification process must be updated to support the new authentication flow.

## Detailed Breakdown

### Registration Process

Replace the current registration process that distributes a temporary password with a workflow that sends a six-digit verification code. The registration email should continue to include the user's registered email address, but the temporary password should no longer be included.

### Account Verification

After users click the **Verify Account** button in the registration email, redirect them to a dedicated verification page instead of immediately completing account verification. The page should prompt users to enter only the six-digit verification code that was sent to their registered email address.

### Verification User Experience

Present the verification code input as six individual fields arranged consecutively. As users enter each digit, the cursor should automatically move to the next field, allowing the entire code to be entered efficiently before selecting **Verify My Account**.

### Post-Verification Process

After the verification code is successfully validated, redirect users directly to the existing forced password change page. The forced password change functionality should remain unchanged, with only the preceding verification method being replaced.

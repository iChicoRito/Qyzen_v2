# Objective
## Improve Notification Actions and Implement Soft Delete for Scores

## Description
Resolve issues affecting the educator experience in the notification and score management modules. Fix the notification action state so loading indicators correctly stop after the requested operation completes. Replace permanent score deletion with a soft delete mechanism that archives deleted records instead of removing them immediately. Streamline the deletion workflow by removing the confirmation dialog and replacing it with an undo-enabled toast notification, while introducing a dedicated archived scores section where educators can restore previously deleted scores.

## Primary Objective
Improve the reliability and usability of notification actions while replacing permanent score deletion with a recoverable soft delete and archive workflow.

## Secondary Objectives
- Fix the loading state of notification action buttons after completing bulk operations.
- Implement a soft delete system that archives deleted scores instead of permanently removing them.
- Replace the deletion confirmation dialog with an undo-enabled toast notification.
- Organize the Scores navigation by separating active and archived scores.

## Supporting Tasks

### Notification Actions
- [Parallel] Fix the **Mark All as Read** action so its loading spinner stops immediately after the operation completes successfully.
- [Parallel] Fix the **Delete All** action so its loading spinner stops immediately after the operation completes successfully.
- Ensure both buttons correctly reset to their normal state after the request finishes.

### Score Soft Delete
- [Parallel] Replace permanent deletion with a soft delete mechanism that moves deleted scores into an archived state.
- Remove the deletion confirmation dialog from the educator workflow.
- After a score is deleted, display a toast notification containing an **Undo** action with a **5-second** countdown.
- If the educator selects **Undo** within the allotted time, restore the score immediately.
- If no undo action is taken within **5 seconds**, move the score to the archived scores list.

### Navigation
- [Parallel] Convert the **Scores** sidebar menu into a dropdown menu.
- Add two submenu items:
  - **Scores** (active scores)
  - **Deleted Scores** (archived scores)

### Archived Scores Management
- [Parallel] Create a **Deleted Scores** page for archived score records.
- Allow educators to restore archived scores back to the active scores list.

## Multi-Agent Feasibility Assessment

**Assessment:** **YES** — The requested enhancements involve multiple independent work streams across notifications, score deletion logic, navigation, and archived score management, making them well suited for parallel implementation.

**Parallelization Opportunities:**
- Fix notification loading state behavior.
- Implement the soft delete backend and deletion workflow.
- Develop the undo toast notification with timer functionality.
- Update the sidebar navigation structure.
- Build the Deleted Scores page and restore functionality.

**Dependencies to Manage:**
- The soft delete implementation must be completed before the Deleted Scores page and restore functionality can be fully integrated.
- The undo toast must interact correctly with the soft delete state before records are finalized as archived.
- Navigation updates should link to the completed archived scores module.

**Named Sub-Agent Assignments:**
- **Agent 1** (Notification State Specialist): Fix the loading spinner behavior for **Mark All as Read** and **Delete All** actions.
- **Agent 2** (Data Management Specialist): Implement the soft delete mechanism, archive workflow, and restore functionality.
- **Agent 3** (UI/UX Specialist): Replace the confirmation dialog with a 5-second undo toast notification and integrate the timer behavior.
- **Agent 4** (Navigation Specialist): Convert the Scores sidebar into a dropdown and implement the **Scores** and **Deleted Scores** submenu structure.

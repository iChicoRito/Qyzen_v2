# Task 21 - Assessment Duplication and Single-Section Editing

Summary of the implementation for `prompts/tasks-assignment/21.md`: educators can duplicate an
existing assessment from the assessment action menu, adjust the duplicate's details in a modal, and
create a separate assessment that keeps the original question pool. Assessment editing now accepts
one subject/section only.

---

## Plain-language summary

Before this change, an educator who wanted to reuse an assessment had to create another one by hand
and rebuild its question pool. The edit form also allowed selecting extra subjects, which quietly
created additional assessments instead of simply editing the one currently open.

Now each assessment has a **Duplicate** action. Clicking it opens a modal filled with the original
assessment details, with the assessment name defaulted to `Copy` so saving immediately avoids a
duplicate-code conflict. The educator can change the duplicate's schedule, rules, status, term, and
single subject/section before saving.

Saving the duplicate creates a new assessment record. The original assessment is not modified. The
new assessment receives the same eligible question pool as the original, so the educator does not
need to reselect questions.

The edit form now changes only the current assessment. It allows one subject/section choice, and
switching that choice updates the same assessment row instead of creating another assessment.

## Technical summary

### Assessment duplication

- Added `GET educator/assessments/{assessment}/duplicate` and
  `POST educator/assessments/{assessment}/duplicate` before the resource route in `routes/web.php`.
- Added `AssessmentController::duplicate()` to render a modal-friendly duplicate form.
- Added `AssessmentController::storeDuplicate()` to create the duplicate inside a transaction.
- The duplicate uses the submitted assessment details, logged-in educator id, selected subject id,
  derived section id, and the source assessment's `pool_size`.
- The duplicate copies question-pool membership with
  `eligibleQuizzes()->sync($source->eligibleQuizzes()->pluck('tbl_quizzes.id'))`.
- If the duplicate is created as active, it sends the same enrolled-student publish notification as
  normal assessment creation.

### Single-section editing

- `UpdateAssessmentRequest` now requires `subject_ids` to be an array with exactly one value.
- `AssessmentController::update()` now updates the current assessment row only.
- The selected subject's `sections_id` is used as the assessment `section_id`.
- The old edit behavior that treated additional selected subjects as new assessments was removed.

### UI changes

- Added a **Duplicate** item to the educator assessment context menu.
- Added `resources/views/educator/assessments/duplicate.blade.php`, reusing the shared assessment
  fields partial.
- Updated `resources/views/educator/assessments/_fields.blade.php`:
  - create mode keeps multi-subject checkboxes.
  - edit and duplicate modes use a single radio-style subject selection.

### Tests and verification

### Note

No schema changes, new dependencies, or poll-specific cloning were added. In the current codebase,
assessment questions are attached through the question-bank pool pivot, so duplication copies those
links instead of cloning shared question rows.

# Task 25 - Notification Actions and Score Archive

Summary of the implementation for `prompts/tasks-assignment/25.md`: made score deletion immediate
and recoverable, added educator-owned deleted-score management, and fixed notification bulk-action
loading cleanup.

---

## Plain-language summary

Educators can now delete a score directly from the Scores table. The attempt is archived immediately
without asking for a password, disappears from active score results, and remains recoverable. A
five-second Undo action appears in a toast after deletion.

The new Deleted Scores destination is available under Scores in the educator sidebar. It shows the
student, assessment, subject, section, result, and deletion time, and allows the owning educator to
restore an archived attempt. Educators cannot access or restore another educator's scores.

Notification Mark All as Read and Delete All actions now always remove their loading state, re-enable
their buttons, and clear `aria-busy`, including when the request fails.

---

## Technical summary

### Score archive and restore

- Added `deleted_at` to `tbl_scores` through the generated soft-delete migration and applied
  Eloquent `SoftDeletes` to `App\Models\Score`.
- Removed the password-confirmation delete route, controller method, validation, and modal view.
- The educator DELETE endpoint soft-deletes immediately and returns JSON containing the restore URL
  for the custom table UI.
- Added `ScorePolicy::restore` and educator-only deleted/restore routes. Route model binding includes
  trashed rows so authorization still runs for cross-owner archived attempts.
- Added the owner-scoped Deleted Scores page with metadata and native Restore forms.

### Query safeguards

- Added explicit `deleted_at IS NULL` predicates to educator and student correlated latest-attempt,
  best-score, and attempt-count subqueries. This is required because raw correlated subqueries do
  not receive Eloquent's global soft-delete scope automatically.
- Deleting the newest attempt therefore exposes the previous active attempt and recalculates the
  visible best score and count correctly.

### UI and navigation

- Replaced the modal delete action with a direct DELETE request.
- Removed the row immediately after success and showed a five-second KTToast with a visible
  `Undo (5)` countdown action that PATCHes the restore endpoint.
- Extended the shared sidebar renderer with KTUI accordion children and configured Scores with
  Scores and Deleted Scores destinations.
- Added local notification loading cleanup in `finally` for both bulk actions.

### Tests and verification

- Added coverage for password-free deletion, soft-delete preservation, active-query hiding,
  cross-owner denial, owner-scoped archive listing, restore, newest-attempt deletion behavior, and
  notification spinner cleanup.

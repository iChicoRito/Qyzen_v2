# Task 26 - Question Bank Archive, Silent Publish, and Inactive-Term Visibility

Summary of the implementation for `prompts/tasks-assignment/26.md`: added batch-safe archived
question management for educators, renamed the deleted-score UX to archived scores, added a silent
publish option for assessments, and hid inactive-term data across educator and student portals
without changing admin academic-term visibility.

---

## Plain-language summary

Educators now have a proper **Archived Questions** flow for the question bank. Archiving works by
batch instead of by one row at a time: selecting any question from a batch archives the whole batch
for that educator, and restoring from the archived page brings the whole batch back. Older
single-question edit/delete behavior stays in place, but the new archive flow is the safe way to
manage question sets that belong together.

The old **Deleted Scores** wording is now **Archived Scores** in the sidebar and archived-score
screen, but the recovery behavior stays the same. Educators can still restore archived score
attempts without a page-reload regression.

Assessment publishing now has three choices instead of a plain active/inactive toggle:
**Inactive**, **Active and notify students**, and **Active without notifying students**. The
assessment still stores a normal active flag in the database; the new publish choice only affects
whether student notifications are sent during that save.

Inactive academic terms are now hidden from educator and student portals. Their sections, subjects,
assessments, quizzes, scores, learning materials, announcements, filters, and related detail pages
drop out of normal portal access until the term is reactivated. Admin academic-term pages were left
alone so admins can still see and reactivate inactive terms.

## Technical summary

### Question bank archive and restore

- Added educator archive routes for question-bank batch archiving and restoring in
  [routes/web.php](/D:/Personal%20Files/Projects/WebApps/qyzen_v2/routes/web.php).
- Added `QuizController::archive()`, `archived()`, and `restoreArchived()` in
  [app/Http/Controllers/Educator/QuizController.php](/D:/Personal%20Files/Projects/WebApps/qyzen_v2/app/Http/Controllers/Educator/QuizController.php).
- Batch identity reuses `batch_label`; null labels are treated as one-row batches through a local
  computed key, so older data stays archivable without a schema change.
- Archive resolves the selected visible quiz rows to educator-owned batch keys, then soft-deletes
  every visible row in those batches inside one transaction.
- Restore loads archived educator-owned rows, groups them by batch key, and restores the selected
  archived batches in one transaction.
- Added the new archived questions page at
  [resources/views/educator/quizzes/archived.blade.php](/D:/Personal%20Files/Projects/WebApps/qyzen_v2/resources/views/educator/quizzes/archived.blade.php)
  and updated the active question-bank screen in
  [resources/views/educator/quizzes/index.blade.php](/D:/Personal%20Files/Projects/WebApps/qyzen_v2/resources/views/educator/quizzes/index.blade.php)
  to submit batch archive selections.

### Archived scores UX

- Renamed educator navigation and page copy from `Deleted Scores` to `Archived Scores` in
  [resources/views/educator/layout.blade.php](/D:/Personal%20Files/Projects/WebApps/qyzen_v2/resources/views/educator/layout.blade.php),

- Reused the existing score soft-delete/restore mechanics instead of adding a second archive
  system.

### Silent publish mode

- Replaced the old raw active select semantics with transient `publish_mode` handling in
  [app/Http/Requests/StoreAssessmentRequest.php](/D:/Personal%20Files/Projects/WebApps/qyzen_v2/app/Http/Requests/StoreAssessmentRequest.php)
  and [resources/views/educator/assessments/_fields.blade.php](/D:/Personal%20Files/Projects/WebApps/qyzen_v2/resources/views/educator/assessments/_fields.blade.php).
- `publish_mode` normalizes back to the stored `is_active` boolean during validation, with a
  request helper deciding whether student notifications should be emitted for that save.
- Assessment create, duplicate, and update paths in
  [app/Http/Controllers/Educator/AssessmentController.php](/D:/Personal%20Files/Projects/WebApps/qyzen_v2/app/Http/Controllers/Educator/AssessmentController.php)
  now notify only when the save is not explicitly silent.
- Inactive-to-active saves still emit `assessment_created` when the chosen mode is notify; silent
  publish skips that notification while still publishing immediately.

### Inactive-term visibility and authorization

- Updated shared `visibleTo()` scopes so educator/student portal queries ignore inactive terms for:
  `Section`, `Subject`, `Assessment`, `Quiz`, `Announcement`, `LearningMaterial`, `Enrolled`, and
  `Score`.
- Tightened `QuizPolicy` and `ScorePolicy` so direct detail routes respect the same visibility rules
  instead of relying only on ownership checks.
- Updated educator/student controllers and filter sources to use `visibleTo()` and active-only term
  lists, covering lists, detail pages, exports, uploads, monitoring, and form dropdowns.
- Qualified shared scope columns such as `educator_id`, `student_id`, and `is_active` at the model
  layer so joined table screens keep working after the visibility scope is applied.

# Objective
## Hint System Redesign: Replace Instant Answer with Game-Based Challenges

---

## Description
The current hint feature on the student quiz page reveals the correct answer immediately upon a single click, making it overpowered. This redesign transforms that interaction into a modal-based mini-game experience. Instead of directly seeing the answer, the student must successfully complete a short challenge—such as solving a puzzle or winning a game—to earn the hint. Failure, timeout, or choosing to skip the challenge consumes a hint credit without revealing any answer. The modal overlays the quiz page, ensuring the student never leaves the quiz context. The goal is to make hint usage more deliberate, engaging, and balanced.

---

## Primary Objective
Replace the one-click answer reveal with a modal mini-game system where the student must win a randomly selected challenge to view the correct answer, deducting exactly one hint credit per attempt regardless of outcome.

---

## Secondary Objectives
- Implement a tic-tac-toe game against a medium-difficulty AI; winning once displays the answer.
- Implement a maze escape game where navigating from start to exit reveals the answer.
- Implement a math riddle challenge that requires entering the correct answer to a basic arithmetic question.
- Implement a word scramble (anagram) challenge using a scrambled version of the answer or a related keyword, with only one unscramble attempt.
- Implement a Simon Says memory game requiring completion of five correct sequences; any wrong click fails immediately.
- Implement a Whack-a-Mole game where the student must hit five moles within 15 seconds, failing if three misses occur or time expires.
- Implement a Color Match (Stroop test) challenge requiring five consecutive correct color identifications; one mistake fails instantly.
- Provide a clearly labeled skip button that allows the student to forfeit the challenge, consuming the hint without revealing the answer.

---

## Success Criteria
- Clicking the hint button opens a modal containing one randomly selected challenge, overlaying the quiz page without navigation.
- Winning a challenge immediately displays the correct answer.
- Failing any challenge (by losing, running out of time, or making a wrong move) consumes one hint credit and does not reveal the answer.
- Pressing the skip button consumes one hint credit and closes the modal with no answer shown.
- The tic-tac-toe opponent operates at a medium skill level, neither too easy nor unbeatable.
- Word scramble accepts only one input attempt and evaluates it instantly.
- Simon Says demands exact replication of a five-round sequence; the first mis-press ends the challenge.
- Whack-a-Mole enforces a 15-second timer and a maximum of three allowed misses; exceeding either fails the challenge.
- Color Match requires five correct responses in a row; the first incorrect selection fails the challenge.
- Every hint interaction, regardless of outcome, reduces the student’s available hint balance by exactly one.

---

## Constraints
- The hint modal must never redirect or reload the quiz page; it appears as an overlay on the current view.
- Tic-tac-toe AI difficulty must be tuned to medium, avoiding overpowered or trivial behavior.
- Word scramble gives the student exactly one attempt; there is no retry.
- Simon Says is fixed at exactly five rounds.
- Whack-a-Mole time limit is exactly 15 seconds, and the miss allowance is exactly three.
- Color Match requires five consecutive correct answers with no room for error.

---

## Context & Dependencies
- The existing student quiz page already includes a hint button that instantly shows the correct answer; this button and the hint credit tracking will be modified.
- The quiz page infrastructure remains unchanged; the modal will be injected as a front-end overlay.

---

## Stakeholders
- Students (the end users who will interact with the redesigned hint modal)

---

## Detailed Breakdown

### Tic-Tac-Toe Challenge
The modal presents a 3×3 grid where the student plays against a medium-difficulty computer opponent. The student must win one game; upon victory, the correct answer is displayed immediately. A loss or draw consumes the hint. The AI is tuned to provide a fair but beatable challenge.

### Maze Escape Challenge
A simple maze is generated with a defined start and end point. The student navigates a character through the maze. Reaching the exit successfully reveals the answer. If the student quits or fails to complete the maze (implicitly by skipping or closing the modal), the hint is consumed.

### Math Riddle Challenge
A basic arithmetic question (addition, subtraction, multiplication, or division) is displayed with an input field. The student must enter the numerically correct answer. A correct submission reveals the quiz answer; an incorrect submission consumes the hint without revealing it.

### Word Scramble (Anagram) Challenge
The modal shows a scrambled version of the correct quiz answer or a closely related keyword. The student must type the unscrambled word into a single input field and submit. Only one attempt is allowed. If the unscrambled word matches, the answer is revealed; any mismatch or empty submission consumes the hint.

### Simon Says (Memory Game) Challenge
A sequence of colored buttons lights up in a pattern, starting with one and growing by one each round up to five rounds. The student must replicate the entire sequence by pressing the buttons in the correct order. On the first wrong press, the challenge ends immediately, and the hint is consumed. Successful completion of all five rounds reveals the answer.

### Whack-a-Mole Challenge
Moles (or targets) appear randomly in different positions within the modal. The student must tap or click a total of five moles. A 15-second countdown is active. If the student misses three moles (clicks an area with no mole) or the timer reaches zero before five moles are hit, the challenge fails and the hint is consumed. Hitting five moles within the limits reveals the answer.

### Color Match (Stroop Test) Challenge
A series of color name words (e.g., “RED”) is shown, each rendered in a font color that may differ from the word’s meaning. The student must select the font color, not read the word. The challenge requires five consecutive correct choices. One wrong selection fails the challenge immediately and consumes the hint. Five correct choices in a row reveal the answer.

### Skip Functionality
A visible skip button is present in every challenge modal. Pressing it immediately forfeits the challenge, deducts one hint credit, and closes the modal without revealing any part of the answer.

# Objective
## Education Platform UI/UX Refinements

---

## Role
You are a full-stack developer and UI/UX specialist. Approach these tasks with a focus on usability, responsiveness, data visualization, and front-end performance. Prioritize clean, maintainable component architecture and an intuitive end-user experience across both student and educator interfaces.

---

## Description
This objective covers a set of targeted improvements to an existing education platform. The work is divided across three task groups: student-facing features, educator-facing features, and shared UI/UX elements. All three agents must work concurrently and independently, with no linear dependencies between their task sets. The student-side updates introduce a new sidebar menu for enrolled subjects, clean up the dashboard, fix mobile calendar responsiveness, and declutter the scores page. The educator-side updates address dashboard layout issues, implement a new heatmap chart, fix mobile calendar responsiveness, add enrollment timeline management, refine button labels, optimize the questions bank data structure, and declutter the scores page. The shared UI/UX tasks make the notifications indicator dynamic and add context indicators to the educator chat view. The overall goal is to enhance usability, streamline data presentation, and fix responsive layout defects through parallel execution.

---

## Primary Objective
Deliver a refined, responsive, and organized user interface for both student and educator roles by implementing specified menu additions, component removals, layout fixes, data organization improvements, dynamic indicators, and data optimization strategies, with all three agents working simultaneously and independently.

---

## Secondary Objectives
- Add an "Enrolled Subject" sidebar menu on the student side displaying a table of subjects with the educator column first and a profile icon.
- Remove the Recent Grades card from the student dashboard.
- Fix the mobile calendar viewport on both student and educator sides to prevent horizontal scrolling and ensure responsiveness.
- Reorganize the scores page on both student and educator sides to reduce visual clutter from multiple assessment attempts.
- Make the notifications drawer indicator dynamic, showing a red dot only when new notifications exist, and hiding it when the tab is checked.
- Add subject/section context indicators to the educator chat interface.
- Fix the Average Score by Section display on the educator dashboard.
- Remove the This Week and Notifications cards from the educator dashboard.
- Implement an APEX-based heatmap chart on the educator dashboard, based on assessments created.
- Add a "Clear History" button to the enrollment timeline to delete all recent reports and associated database records.
- Shorten the "Unenroll All Students" context menu label to "Un-enroll".
- Optimize the questions bank data structure to keep it clean and reduce database size growth.

---

## Success Criteria
- The "Enrolled Subject" menu item appears in the student sidebar and renders a table with subject, section, and educator columns; the educator column is positioned first and includes a profile icon.
- Recent Grades card is no longer visible on the student dashboard.
- The calendar page renders without horizontal scrolling on mobile viewports for both student and educator roles.
- The scores page presents multiple attempts per assessment in an organized, non-cluttered layout for both roles.
- The notifications drawer indicator is red and appears only when new, unseen notifications are present; it hides after the respective tab is viewed.
- The educator chat displays clear subject and section context for each conversation.
- The Average Score by Section on the educator dashboard renders correctly without formatting issues.
- The This Week and Notifications cards are removed from the educator dashboard.
- A functional APEX heatmap chart appears on the educator dashboard, populated based on created assessments.
- A "Clear History" button exists on the enrollment timeline and successfully deletes all recent upload records and their database entries.
- The context menu option reads "Un-enroll" instead of "Unenroll All Students".
- The questions bank uses an optimized data structure that demonstrably reduces data bloat.

---

## Constraints
- Agents 1, 2, and 3 must operate concurrently and independently; the workflow is not linear and no agent should block another's progress.
- Use the shared table component for the Enrolled Subject table.
- Use APEX Charts for the educator heatmap implementation.
- The questions bank optimization must reduce database size, but the exact approach requires further discussion if suggestions are needed.

---

## Context & Dependencies
- An existing education platform with student and educator interfaces, sidebar navigation, a shared table component, and APEX Charts integration.
- A notifications drawer with "All" and "Inbox" tabs currently using a fixed green dot indicator.
- A calendar page currently suffering from a non-fixed, scrollable viewport width on mobile.
- A scores page currently displaying all assessment attempts in a crowded manner.
- An educator dashboard with an Average Score by Section section, a This Week card, and a Notifications card.
- An enrollment timeline feature labeled "Recent enrollment uploads."
- A context menu containing the action "Unenroll All Students."
- An existing questions bank that is creating increasing database size as educators add questions.
- All three agents will work on their respective task sets simultaneously with no sequential dependencies between agents.

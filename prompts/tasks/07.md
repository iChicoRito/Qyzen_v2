# Objective
## Optimize Platform Performance and User Experience Across Multiple System Components

---

## Role

You are a full-stack development team lead coordinating parallel fixes across a deployed web application. Approach this objective with a focus on system stability, performance optimization, and responsive design improvements.

---

## Description

This objective addresses four distinct technical issues identified in a live deployment environment. The platform requires fixes for persistent asset loss during Git deployments, notification system performance concerns, file download failures on the student materials interface, and responsive layout improvements for mobile dashboard views. These tasks are to be executed independently and simultaneously by separate sub-agents rather than in a linear workflow sequence. Each agent will own a specific component of the system and deliver targeted solutions without dependency on other agents' workstreams.

---

## Primary Objective

Execute four independent, parallel remediation tasks to resolve critical system issues related to asset persistence, notification management, file accessibility, and mobile interface responsiveness.

---

## Secondary Objectives

- Investigate and resolve the root cause of profile picture and cover photo loss occurring after Git pushes to the Hostinger deployment repository
- Assess notification table management and propose a strategy for automatic deletion or archival to prevent performance degradation
- Diagnose and fix the 404 file download error encountered on the student materials page
- Reconfigure dashboard KPI card layout from 2 columns to a single column on mobile viewports to eliminate ellipsis truncation

---

## Success Criteria

- Profile pictures and cover photos persist through Git deployment cycles without manual restoration
- A documented solution for notification cleanup is proposed, reviewed with the requester, and ready for implementation
- File downloads from the student materials section complete successfully without 404 errors
- Dashboard KPI cards display as a single column on mobile devices with full text visibility

---

## Constraints

- All tasks must be completed independently and simultaneously by separate sub-agents without sequential dependencies
- Notification solution requires requester confirmation before implementation

## Detailed Breakdown

### Asset Loss During Deployment
The deployed system currently loses user-uploaded profile pictures and cover photos whenever a new Git push is made. Because Hostinger is connected directly to the repository, any push triggers an automatic deployment that appears to overwrite or omit stored image assets. The assigned agent must investigate whether these assets are stored within the repository itself, in a separate storage location that is not persisted, or if dependencies are missing that restore these files post-deployment. The solution should ensure that images remain intact through deployment cycles without requiring manual re-upload.

### Notification Table Management
The notification system stores data in a database table that may grow significantly over time, potentially causing performance slowdowns. The agent must first determine if the notification system currently includes functionality to automatically delete or archive old records. If no such feature exists, the agent should formulate a proposed approach—such as scheduled deletions, archival to a separate table, or time-based retention policies—and present this solution to the requester for confirmation before proceeding with any code changes.

### Student Materials File Download Failure
Students are experiencing 404 errors when attempting to download attached files from educators, with the error appearing on endpoints structured like `/student/materials/21/download`. The assigned agent must investigate whether the issue stems from incorrect file path resolution, missing or moved files, permission errors, or routing misconfigurations. The investigation should trace the request lifecycle from the student interface through the server-side file retrieval logic to identify the specific point of failure and implement a corrective fix.

### Mobile Dashboard KPI Layout
The dashboard KPI cards currently display in a 2-column grid layout on mobile viewports, which causes text truncation and ellipsis overflow. The assigned agent must modify the responsive CSS grid or flexbox configuration to render these cards as a single column on mobile devices. The change should preserve readability and ensure all KPI information is fully visible without horizontal scrolling or content clipping.
